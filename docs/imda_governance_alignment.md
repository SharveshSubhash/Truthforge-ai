## Traceability Matrix

### Principle 1 — Internal Governance Structures and Measures

> *Organisations should establish clear accountability and governance over AI systems.*

| IMDA Requirement | Implementation in TRUTHFORGE AI | Code Evidence |
|---|---|---|
| Designate roles and responsibilities for AI oversight | Each agent has an explicit `AGENT_VERSION` constant and `source_agent` field in `AgentStatus` — every decision is attributed to a named agent | `agents/consistency_analysis_agent.py:60`, `core/state.py` |
| Maintain documentation of AI model decisions | `artifacts/runs/*.json` — every pipeline run produces a persistent JSON record with model name, transcript hash, inconsistency count, duration | `core/run_metadata.py` |
| Track model versions for reproducibility | `PROMPT_VERSION = "v1.1"` in explainability agent; `AGENT_VERSION = "v1.0"` in consistency agent; saved per-run to memory store | `agents/explainability_agent.py:104`, `agents/consistency_analysis_agent.py:60` |
| Audit trail for AI-augmented decisions | `logs/events.jsonl` — append-only structured log of every pipeline event, security decision, and bias flag | `core/logger.py`, `core/metrics.py` |

### Principle 2 — Determining the Level of Human Involvement in AI-Augmented Decision-Making

> *Organisations should determine how much human oversight is required based on the stakes and context of AI decisions.*

| IMDA Requirement | Implementation in TRUTHFORGE AI | Code Evidence |
|---|---|---|
| Human oversight proportional to risk | Legal transcript analysis is HIGH risk — final report includes mandatory disclaimer: *"does not constitute legal advice… Human legal professionals retain full responsibility"* | `agents/explainability_agent.py` `_build_clean_report()` |
| Flag uncertain AI decisions for human review | `requires_review=True` flag triggers a second-pass re-analysis when consistency findings are low-confidence; surfaced in UI | `agents/consistency_analysis_agent.py` `consistency_analysis_node()` |
| Humans can override or reject AI output | Streamlit UI presents all findings as analytical input — no automated action is taken; legal professionals make final decisions | `ui/results.py`, report disclaimer |
| AI system must not make final legal determinations | System prompt explicitly forbids: *"Do NOT draw conclusions about guilt, innocence, or credibility"*; output gate enforces this by redacting verdict language | `agents/consistency_analysis_agent.py:163–166`, `agents/responsible_ai_security_agent.py:98–105` |

### Principle 3 — Operations Management

> *Organisations should implement robust processes to manage AI system performance and risk in production.*

| IMDA Requirement | Implementation in TRUTHFORGE AI | Code Evidence |
|---|---|---|
| Monitor AI system performance in production | `core/metrics.py` — tracks `pipeline_runs_total`, `pipeline_failures_total`, `avg_runtime_ms`, `pipeline_blocked_total` with rolling 100-run window | `core/metrics.py:MetricsCollector` |
| Alert on anomalous behaviour | `_check_alerts()` emits `RuntimeWarning` when failure rate or security blocks exceed thresholds (`FAILURE_THRESHOLD=5`, `BLOCK_SPIKE_THRESH=3`) | `core/metrics.py:_check_alerts()` |
| Graceful degradation when AI components fail | All agents fall back to deterministic rule-based analysis if LLM call fails or is unavailable — pipeline completes without LLM | `agents/consistency_analysis_agent.py:_rule_based_analyse()`, `agents/explainability_agent.py:_fallback_explain()` |
| Input validation before AI processing | Security input gate scores every transcript for adversarial content before any LLM processes it; blocks if score ≥ 0.5 | `agents/responsible_ai_security_agent.py:validate_input()` |
| PII and sensitive data controls | Output gate redacts Singapore NRIC/FIN, passport numbers, phone numbers, bank accounts, email addresses before report delivery | `agents/responsible_ai_security_agent.py:_PII_PATTERNS` |

### Principle 4 — Stakeholder Interaction and Communication

> *Organisations should be transparent with stakeholders about how AI is used and what its limitations are.*

| IMDA Requirement | Implementation in TRUTHFORGE AI | Code Evidence |
|---|---|---|
| Disclose AI involvement to end users | Mandatory report footer: *"This report is generated by TRUTHFORGE AI as an analytical aid. It does not constitute legal advice."* — cannot be removed by configuration | `agents/explainability_agent.py:_build_clean_report()` |
| Explain how AI reached its conclusions | ReAct reasoning chain (`observe` → `reason` → `explain` → `recommend`) — every explanation step is a separate auditable field, not a black box | `agents/explainability_agent.py:ExplanationModel` |
| Communicate confidence levels | `confidence` field (HIGH/MEDIUM/LOW) on every inconsistency and explanation, with rubric defined in code (`CONFIDENCE_RUBRIC`) | `agents/explainability_agent.py:125–138`, `agents/consistency_analysis_agent.py:65–70` |
| Communicate system limitations | `analysis_notes` field from LLM captures self-reported limitations per run (e.g. transcript ambiguity, missing context) | `agents/consistency_analysis_agent.py:ConsistencyReportModel` |

### Principle 5 — Fairness and Non-Discrimination

> *AI systems should treat all individuals fairly and must not discriminate based on protected characteristics.*

| IMDA Requirement | Implementation in TRUTHFORGE AI | Code Evidence |
|---|---|---|
| Equal treatment regardless of identity | System prompts mandate: *"Apply equal scrutiny to prosecution, defence, and all witnesses regardless of which side they support"* | `agents/consistency_analysis_agent.py:169`, `agents/explainability_agent.py:_SYSTEM_PROMPT` |
| Prohibited characteristics not used in decisions | `_BIAS_PATTERNS` — 7 regex patterns blocking race, religion, and gender-based credibility generalisations in output | `agents/responsible_ai_security_agent.py:109–126` |
| Neutral language enforced | Forbidden: "the victim", "the perpetrator" — system prompts require: "the witness", "the accused", "the complainant" | `agents/consistency_analysis_agent.py:171–172` |
| Post-generation bias verification | `_check_output_bias()` scans LLM output for `_UNSAFE_PHRASES` after generation — independent of prompt guidance | `agents/explainability_agent.py:288–315` |
| Bias tests in CI pipeline | `tests/security/test_fairness_bias.py` — automated tests verify bias patterns are caught and rejected | `tests/security/test_fairness_bias.py` |

### Principle 7 — Security

> *AI systems must be resilient against adversarial inputs and protect against misuse.*

| IMDA Requirement | Implementation in TRUTHFORGE AI | Code Evidence |
|---|---|---|
| Protect against prompt injection | 3-tier detection: 12 definitive patterns (0.6 score each) + 9 ambiguous patterns (0.3) + role confusion heuristics (0.4) | `agents/responsible_ai_security_agent.py:43–82` |
| Input sanitisation | Unicode normalisation (NFC), control character removal, whitespace normalisation — applied before LLM processing | `agents/responsible_ai_security_agent.py:_sanitize()` |
| Adversarial robustness testing | 15+ injection payloads tested in `tests/security/test_prompt_injection.py` including DAN, jailbreak, persona replacement, system tag injection | `tests/security/test_prompt_injection.py` |
| Constrained output prevents instruction-following | Pydantic structured output on all LLM calls — model cannot output free-text instructions that downstream code would execute | All agent `with_structured_output()` calls |
| Security event logging | All injection attempts, bias detections, PII findings, and neutrality violations logged to `logs/events.jsonl` with timestamp | `core/metrics.py:record_security_event()` |
